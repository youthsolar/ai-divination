# PRD — AI占卜（易經/塔羅/符令）+ 老師諮詢預約 + 法器/法術商品

> 本文件是本專案想法與需求的「單一入口」。Jeffery 可直接把任何想法丟到 INBOX，我會定期整理成可執行規格並同步到 ROADMAP。

## INBOX（先丟想法，整理由我負責）
- 目標調整：每日 100 筆成交購買符令。
- 產品入口：LINE 免費占卜（易經/塔羅）作為吸引；使用者可選擇
  - 直接買數位符令（數位福利）解決問題
  - 或依問題取向，預約「真實通靈老師」到現場/線上諮詢（用 SimplyBook 類型系統）
  - 或購買法器/法術類商品（目前用 EasyStore 類型系統銷售）
- 限制：n8n CE 執行紀錄僅保存 1 天 → 金流/交付/對帳必須在 Zoho Creator 做 SSOT。
- 交付：付款後要能透過 LINE 交付符令圖片（交付策略待部署前決策）。
- Phase 2 構想（SimplyBook）：n8n 定期同步老師/預約項目；當使用者不買數位符令或表達「很麻煩/很急」等訊號時，直接提供預約方向與連結（可先做單一入口連結，再進階做個人化推薦）。

## 核心目標（Draft）
- North Star：100 單/天（符令）
- 轉換策略：免費占卜 → 付費買符令（新客為主）
- 第二/三階段擴展：導向「老師諮詢預約」與「法器/法術商品」的加購/升級路徑

## 使用者旅程（Draft）
1) LINE 免費占卜（生日+問題）
2) 取得免費結果（安撫+方向）
3) CTA：購買符令（數位福利）
4) 付款成功：LINE push 交付（連結/圖片）
5) 升級路徑：
   - 需要深度/真人介入 → 推薦預約老師（SimplyBook）
   - 需要實體/進階法器 → 推薦商品（EasyStore）

## 產品模組（Draft）
- Divination：易經 + 塔羅（MVP）
- Talisman：符令（數位交付）
- Payment：ECPay（信用卡）
- Delivery：token URL / LINE 圖片
- Booking：老師預約（SimplyBook）
- Commerce：法器/法術商品（EasyStore）
