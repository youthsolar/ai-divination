# Phase 1 PRD-lite — LINE 免費占卜 → 付費符令（數位交付）

Owner: Jeffery
Last updated: 2026-02-18

> 目標：跑通「免費占卜 → 付費符令 → LINE 交付 → 冷卻回購」閉環，能小量營運。

## 1) MVP 範圍（In / Out）

### In（必做）
- 入口：LINE（LIFF 或聊天引導皆可）
- 必填輸入：**國曆生日 + 問題**
- 占卜：**塔羅 + 易經**（系統自動分流/或同時產出一份整合結果）
- 冷卻機制：避免情緒連問/濫用（冷卻期間不打模型）
- 付費：ECPay（信用卡）
- SSOT：Zoho Creator 記帳、訂單、交付、冷卻狀態
- 交付：付款成功 → LINE push 交付（token URL / 圖片策略二選一）
- Token 有效期：7 天（到期處理）
- 回拋重送：ECPay callback 重送不造成重複交付

### Out（先不做）
- 真人老師預約（Phase 2）
- 法器/法術商品（Phase 3）
- 複雜命理資料蒐集（農曆、八字、生肖等都由系統推導或不做）

## 2) 使用者旅程（MVP）

1. 使用者在 LINE 進入「免費占卜」
2. 輸入：生日、問題
3. 系統回覆：
   - 先安撫（降低焦慮）
   - 給一份免費結果（方向 + 建議）
   - 告知冷卻原則（建立公信力）
4. CTA：購買符令（NT$360）
5. 付款成功：LINE 收到交付（連結/圖片），可查看內容
6. 冷卻與回購：同題同結果下，**7–14 天**可回來回購下一張（最多 5 張進度）

## 3) 冷卻/回購規則（v1）

### 3.1 名詞
- **QuestionKey**：用來判定「同題」的鍵
- **Series**：同題回購的序列（最多 5 張）

### 3.2 同題判定（MVP 簡化）
- 先採 **嚴格策略**：同一個 LINE userId + 問題全文（去空白/全形半形）hash。
- 後續再做語意近似（Phase 1 不做，以免誤判）。

### 3.3 冷卻窗口
- 建議 v1：固定 **7 天**（先做死，能上線最快）；UI/文案寫「7–14 天」但系統先採 7 天。
- 冷卻期內：
  - 不打 LLM
  - 回覆固定模板：安撫 + 告知下次可占卜/可回購時間 + 引導做一個小行動

### 3.4 回購張數上限
- Series 上限：**5 張**
- 超過：引導真人老師/其他服務（先留文案，不做串接）

## 4) DoD（上線驗收）

### 4.1 一筆端到端測試單
- LINE 送出問題 → 收到免費結果
- 點購買 → ECPay 付款成功
- Creator 記帳成功（訂單狀態=PAID）
- LINE 收到交付（且 token 可開啟內容）

### 4.2 冪等性
- ECPay callback 重送 N 次：
  - Creator 不會產生重複訂單
  - LINE 不會重複交付（或重複交付也必須同一份內容且有去重紀錄）

### 4.3 可營運性
- 有基本對帳欄位（訂單號、金額、付款時間、LINE userId、交付狀態）
- 有基本錯誤告警（callback fail / push fail）

## 5) 需決策的 3 個點（今天就能定）

1) **交付策略**：
- A. Token URL（最簡單、可控、可撤銷）
- B. LINE 圖片直接交付（體驗最好，但要處理檔案保存/權限/重送）

2) **塔羅+易經的呈現**：
- A. 系統自動挑一個（根據問題類型）
- B. 兩個都出，但有一個是主結果（推薦）

3) **冷卻天數**：
- A. 固定 7 天（先上線）
- B. 依類別 7–14 天（之後再做）
